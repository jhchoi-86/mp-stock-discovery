# [v6.6.0] 작업지시서: 랜딩페이지 성과 분석 통합

본 지시서는 작업계획서의 설계 내용을 바탕으로 한 세부 구현 가이드라인입니다.

## 상세 구현 지시

### 1. 백엔드 (server.cjs)
- `getPerformanceSnapshotData` 헬퍼 함수를 사용하여 `/api/public/daily-snapshots` 라우트 구현.
- `PAID` 권한이 없을 경우 `403` 에러를 반환하여 데이터 보호.

### 2. 프론트엔드 API (adminService.js)
- `getPublicSnapshots`, `getPublicSnapshotDates` 메서드 추가.

### 3. 레이아웃 통합 (PcDashboard.jsx, MobileDashboard.jsx)
- `landingTab` 상태를 사용하여 탭에 따른 컴포넌트 스위칭 구현.
- 관리자 패널 활성 여부와 상관없이 내비게이션 탭이 항상 최상단에 노출되도록 배치.
- `DailySnapshotAnalytics.jsx`에서 `isMobile` props에 따른 렌더링 분기 처리.

## 주의사항
- 코드 수정 후 반드시 `update_hash.cjs`를 실행하여 무결성 해시를 갱신할 것.
- 기존의 Admin 전용 대시보드 로직과 충돌이 발생하지 않도록 주의할 것.
