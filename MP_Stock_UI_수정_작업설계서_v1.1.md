## 1. 작업 개요

### 1.1 목적
통합 동기화 실행 후 표시되는 **종목별 신호 및 분석 카드 UI**의 컬럼 구성 및
표시 데이터를 재편하여 가독성·정보 밀도를 개선한다.

## 2. 현재 화면 구조 (AS-IS)

### 2.1 컬럼 헤더
```
종목명 | 세력강도 | 점수 | 이평선 주가 💡 | 매수신호 발생 | 추세 | 추천매매(분할매수전략) | 작업
```

### 2.2 이평선 주가 셀 현재 표시
```
2H 이평 정렬 💡
📍 29,000원       ← 현재가 (상단 고정)
SMA5:   28,340원
SMA10:  25,830원
SMA20:  23,073원
SMA60:  20,059원
```

### 2.3 매수신호 발생 셀 현재 표시
```
[30M] [1H] [2H]        ← 활성 (컬러 버튼)
[4H] [1D] [2D] [1W]    ← 비활성 (회색 버튼)
```
> 현재: 단일 신호 유형(매수)만 표시, 추세·강력 신호 구분 없음

## 3. 수정 후 화면 구조 (TO-BE)

### 3.1 컬럼 헤더 변경

| AS-IS | TO-BE | 비고 |
|-------|-------|------|
| 종목명 | **종목명** | 표시 데이터 구조 변경 (§3.2) |
| 세력강도 / 점수 / 이평선 주가 💡 | **이평선배열(2H)** | 컬럼 통합·재편 |
| 매수신호 발생 / 추세 | **신호발생구간** | 컬럼 통합·재편 |
| 추천매매(분할매수전략) | **추천매매** | 헤더 문자열만 단축, 데이터 수정 없음 |
| 작업 | **작업** | 변경 없음 |

> ⚠️ **세력강도** 컬럼은 종목명 셀 하위 카테고리 뱃지로 흡수되므로 독립 컬럼 제거

---

### 3.2 종목명 컬럼 — 표시 데이터 상세

**레이아웃 (3행 구성)**

```
┌──────────────────────────────────┐
│  동국제약                         │  ← 종목명 (Bold)
│  (KOSDAQ 150 | 086450)           │  ← 시장구분 | 종목코드 (소형, 회색)
│  ★★★★★  90점                  │  ← 별점(5점 만점) + 점수 숫자 (황색 별)
└──────────────────────────────────┘
```

**데이터 매핑**

| 표시 항목 | 현재 데이터 필드 | 형식 |
|-----------|---------------|------|
| 종목명 | `stock.name` | 문자열, Bold |
| 시장구분 | `stock.market` (예: KOSDAQ 150) | 문자열, 소형 회색 |
| 종목코드 | `stock.code` | 6자리 숫자 |
| 별점 | `stock.total_score / 20` 반올림 | ★☆ 아이콘 (5점 만점) |
| 점수 | `stock.total_score` | 숫자 + "점" |

---

### 3.3 이평선배열(2H) 컬럼 — 표시 데이터 상세

**레이아웃 (가격 내림차순 정렬, 5행)**

동국제약 예시 기준:
```
┌──────────────────────────────────┐
│  29,000원  (현재가)  ◀ 강조      │  ← 현재가가 최고가이므로 최상단
│  28,340원  (5일)                 │
│  25,830원  (10일)                │
│  23,073원  (20일)                │
│  20,059원  (60일)                │  ← 최저가 → 최하단
└──────────────────────────────────┘
```

**정렬 규칙**
- 5개 항목(현재가 + SMA 4개)을 **가격 내림차순**으로 정렬
- 현재가 위치는 고정 아님 — 실제 가격 기준으로 정렬 삽입
- 현재가 행은 **Bold + 색상 강조** 처리
- null 값 항목은 필터링 후 `"-"` 표시

---

### 3.4 신호발생구간 컬럼 — 표시 데이터 상세

**레이아웃 (3행 × 7버튼)**

```
┌───────────────────────────────────────────────────────────────┐
│  매수신호  [30M] [1H] [2H] [4H] [1D] [2D] [1W]              │
│  추세신호  [30M] [1H] [2H] [4H] [1D] [2D] [1W]              │
│  강력신호  [30M] [1H] [2H] [4H] [1D] [2D] [1W]              │
└───────────────────────────────────────────────────────────────┘
```

**타임프레임 버튼 규칙**
- 활성 TF: 컬러 버튼 (신호 유형별 색상 구분 권장)
- 비활성 TF: 회색 버튼
- `[30M]` 버튼은 30분봉 신호 활성 여부 기준

## 5. 수정 제외 항목 (명시적 동결)

아래 항목은 **이번 작업에서 절대 수정하지 않는다.**

- 추천매매 컬럼의 실제 데이터 표시 내용 및 계산 로직 (`result_2`, `t1D.bb_upper`)
- 작업 컬럼의 버튼·링크·동작
- `최종 매수 (HH)` 버튼 레이블
- `signal.cond_up7`, `signal.DHH2`, `signal.entry_approved`, `signal.signal_HH` 판정 로직
- `signal.progress`, `signal.adx` 부가 지표 표시
- KIS 실시간 수급 데이터 (`resolvedKisData.*`)
- 거래대금, 외국인/기관 표시 영역
- 전체 카드 레이아웃 그리드·스타일시트 (필요 최소 CSS만 수정 허용)
