# [블루팀] 11대 지표 데이터 통합 및 SSOT 아키텍처 구축 프로젝트 계획서 v1.0

**작성일**: 2026-04-05
**책임자**: 블루팀 (개발 총괄)
**목표**: 플랫폼 전 채널(웹, 텔레그램, DB)의 11대 핵심 지표를 100% 동기화하고 DB 영속성을 확보함.

---

## 1. 기술적 아키텍처 개요
### Single Source of Truth (SSOT) 구조
- **원천 데이터**: NCP(Naver Cloud) MariaDB 내 `KOSPI_STOCKS` 및 `SIGNAL_REPORTS`
- **전파 경로**: 
    1. 관리자 툴/스크래퍼 -> DB 저장
    2. DB -> API 서버 (Node.js) -> 프론트엔드 (React)
    3. DB -> 텔레그램 발송 엔진 (Python/Node.js)

---

## 2. 단계별 마스터 로드맵

### Phase 1: DB 인프라 고도화 (Day 1)
- **스키마 확장**: 11대 지표(거래대금, 추세, 별표 등) 저장용 컬럼 7종 신설.
- **클린 데이터 마이그레이션**: 현재 검증된 `OFFICIAL_TOP5` 데이터를 DB 마스터 테이블에 `Upsert` 하여 과거 오염된 데이터를 정화.

### Phase 2: 통합 데이터 주입 (Day 1-2)
- **API 레이어 개편**: 모든 프론트엔드 요청에 대해 DB의 공인 수치를 래핑하여 응답하는 미들웨어 구축.
- **텔레그램 로직 동기화**: 기존 메시지 포맷(Layout)은 **100% 유지**하되, 내부 템플릿 엔진이 참조하는 값만 DB 필드와 바인딩.

### Phase 3: 프론트엔드 방어막 제거 (Day 2)
- **Hard-coding cleanup**: `officialData.js` 등 임시 상수 파일을 제거하고 완전 자동화된 DB 조회 방식으로 전환.
- **실시간성 검증**: 장중 SSE(실시간 시세)가 정적 데이터 위에서 충돌 없이 구동되도록 레이어 우선순위 확정.

---

## 3. 리스크 관리 포인트
- **데이터 백업**: DB 스키마 수정 전 `mysqldump`를 통한 즉시 복구 지점 확보.
- **API 타임아웃**: 대시보드 리스트 조회 시 조인(Join) 연산 최적화로 렌더링 속도 유지.
