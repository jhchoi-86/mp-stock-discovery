# [작업 지시서 v1.2] 11대 핵심 지표 DB 영속성 및 전 채널 실시간 동기화

**작성일**: 2026-04-05
**버전**: v1.2 (텔레그램 포맷 유지 및 데이터 주입 원칙 반영본)
**상태**: 사용 승인 대기 (PENDING APPROVAL)
**대상**: MP Stock 플랫폼 전 채널 (웹, DB, 텔레그램)

---

## 1. 목적 및 배경
현재 프론트엔드(`v7.7.49`)의 긴급 수치 보정은 하드코딩된 상수에 의존하고 있습니다. 이는 DB 오염으로부터 서비스를 즉시 구제하기 위한 조치였으나, 사용자의 지시에 따라 모든 데이터를 DB에 영속화하고 텔레그램 메시지 포맷까지 하나의 원천(SSOT)으로 통합하는 **근본적인 아키텍처 고도화**를 수행합니다.

---

## 2. 작업 대상 11대 핵심 지표
- **가격 지표 (4개)**: 1차매수가, 2차매수가, 손절가, 목표가
- **수급 지표 (2개)**: 외국인 순매수/도량, 기관 순매수/도량
- **성과 지표 (3개)**: 거래대금, 종목 총점수, 별표(등급)
- **추세 지표 (2개)**: 추세판별(Type), 추세강도(Strength)

---

## 3. 세부 작업 계획 (Step-by-Step)

### [Phase 1] DB 스키마 확장 및 데이터 정제
- **DB 테이블 보강**: `KOSPI_STOCKS` 및 `SIGNAL_REPORTS` 테이블에 거래대금, 추세판별, 추세강도 등의 컬럼을 신설합니다.
- **Master Data Backfilling**: 현재 `officialData.js`에 검증된 11가지 지표를 백엔드 DB로 '역주입'하여 데이터 정합성을 확보합니다.

### [Phase 2] 백엔드 API 및 발송 엔진 통합
- **Unified API Gateway**: 프론트엔드가 DB의 '공인 수치'를 최우선으로 가져오도록 API 응답 로직을 정규화합니다.
- **Telegram Bot 연동**: 텔레그램 발송 스크립트(`telegram_service.js`)가 메시지 생성 시 DB의 11대 지표를 참조하도록 포맷을 일치시킵니다.

### [Phase 3] 프론트엔드 동적 데이터 바인딩
- **Constant 제거**: 하드코딩된 `officialData.js`를 제거하고, 모든 화면이 DB API를 실시간으로 바라보도록 전환합니다.
- **현재가 실시간성 유지**: SSE(실시간 시세) 로직을 유지하여 장중 현재가가 DB 결과 위에 실시간 덮어씌워지게 합니다.

---

## 4. [검증 보고] 레드팀(Red Team) & 블루팀(Blue Team) 교차 검증 결과

### 🏴‍☠️ 레드팀 (Red Team) - 리스크 재검토
- **텔레그램 포맷 유지**: 사용자의 지시에 따라 메시지 레이아웃은 현재 형식을 **100% 보존**합니다. 
- **데이터 소스 주입(Injection)**: 기존 하드코딩 또는 임시 수식을 제거하고, 메시지 생성 시 DB의 11대 지표를 **동적으로 매핑**하여 전달하는 로직으로 교체합니다.
- **정합성 유지**: 웹 플랫폼과 텔레그램 메시지 간의 수치 불일치를 0%로 만들되, 시각적 사용자 경험(UX)은 현재와 동일하게 유지하는 데 집중합니다.

### 🛠 블루팀 (Blue Team) - 기술적 보강 (동일)
- **데이터 타입 정규화**: 수급량(Int), 수익률(Float) 등 DB 필드 타입을 엄밀히 정의하여 연산 오류 원천 차단.
- **동시성 원칙**: 실시간 가격(SSE)은 항상 DB 저장된 정적 가격보다 렌더링 우선순위를 높게 가져갑니다.

## 5. 기대 효과 및 안전 조치
- **데이터 무결성**: 웹, 텔레그램 어디서든 1원의 오차도 없는 동일한 정보가 송출됩니다.
- **휴먼 에러 방지**: DB 한 곳만 관리하면 모든 채널이 자동 동기화되어 오타 사고가 미연에 방지됩니다.
