## 1. 블루팀(Blue Team) 오류 보정 [B-ERR]

### [B-ERR-01] 캐시 레이어 아키텍처 반영
- **보정**: 텔레그램 발송 및 API 조회 성능을 위해 **Redis 캐시**를 'DB'와 '애플리케이션' 사이에 배치한 개선된 아키텍처 다이어그램을 마스터 플랜 v2.0에 수록합니다.
- **경로**: 관리자 툴 -> DB -> **[Redis Cache]** -> API/텔레그램 서비스.

### [B-ERR-02] Phase 의존성 및 순차 실행 명문화
- **보정**: Phase 1(DB 스키마) 완료 화답(Acknowledgement) 수신 후에만 Phase 2(API 연동)를 개시하도록 **Strict Sequential Execution** 원칙을 수립했습니다.

### [B-ERR-03] 롤백 및 스냅샷 시점 전진 배치
- **보정**: `officialData.js` 및 DB 스냅샷 생성 시점을 `Phase 1` 시작 전인 **[Phase 0: Readiness]** 단계로 전진 배치했습니다.

### [B-ERR-04] 11대 핵심 지표 명문화
- **보정**: [가격 4종: 매수1/2, 손절, 목표], [수급 2종: 외인, 기관], [분석 5종: 대금, 점수, 별표, 추세판별, 추세강도]로 명확히 정의했습니다.

### [B-ERR-05] 텔레그램 수정 범위 정의 (Layout vs Data)
- **보정**: 템플릿(HTML/TEXT) 구조는 수정하지 않되, 내부 코드에서 `Value Mapping` 로직만 DB 소스로 전환함을 작업 지시서에 구체화했습니다.

## 2. 레드팀(Red Team) 누락 보정 [R-MISS]

### [R-MISS-01] NCP MariaDB ALTER WAIT 상세 SQL
- **보정**: `SET SESSION lock_wait_timeout = 60; ALTER TABLE ... WAIT 30;` 명령을 사용하여 메타데이터 잠금을 제어하고, 실행 전 `pm2 stop analyzer` 로 실시간 프로세스를 일시 정지하는 시퀀스를 수립했습니다.

### [R-MISS-02] Redis 캐시 무효화(Invalidation) 전략
- **보정**: **Write-Through(동시 갱신)** 전략을 채택하여 DB 수정 즉시 캐시를 갱신하고, 데이터 정합성을 위해 1시간 단위의 `TTL`을 설정합니다.

### [R-MISS-03] 500% 오차 필터 정밀화
- **보정**: 기준점을 **'직전일 종가(Close)'**로 확정하며, 권리락/감자 등 특수 케이스 발생 시 관리자 승인을 통해서만 저장 가능한 `Override` 모드를 도입합니다.

### [R-MISS-04] CheckSum 익일 검수 구체안
- **보정**: 매일 오전 08:30에 DB 수치와 배너/텔레그램 템플릿의 최종 렌더링 수치를 비교하여 불일치 시 관리자 텔레그램으로 즉시 알림을 발송하는 `Audit Scraper` 구현체를 설계했습니다.
