# 💰 [Cost Analysis] AWS EC2 인스턴스 업그레이드 비용 분석

## 1. 지역(Region) 기준
- **Region**: ap-southeast-2 (Sydney) - 현재 RDS 데이터베이스 위치 기준.

## 2. 인스턴스 사양별 비용 비교 (월 730시간 가동 기준)

| 인스턴스 타입 | RAM | CPU | 시간당 비용 | 월 예상 비용 (On-Demand) |
| :--- | :--- | :--- | :--- | :--- |
| **t3.micro (현재)** | 1 GB | 2 vCPU | $0.0132 | **$9.64** (약 13,000원) |
| **t3.medium (변경)** | **4 GB** | 2 vCPU | $0.0416 | **$30.37** (약 42,000원) |
| **t3.large (참고)** | 8 GB | 2 vCPU | $0.0832 | **$60.74** (약 85,000원) |

## 3. 비용 절감 전략 (AWS Reserved Instance)
글로벌 플랫폼(mpstock.co.kr)은 24시간 상시 가동되어야 하므로, 1년 약정(Reserved Instance, No Upfront)을 활용할 경우 비용을 획기적으로 낮출 수 있습니다.

- **t3.medium (1년 약정 시)**: 월 약 **$17.75** (약 25,000원)
- **절감 효과**: On-Demand 대비 약 **41% 할인**.
- **결론**: 현재보다 월 **약 12,000원 정도만 추가**하면 4GB RAM 환경으로 업그레이드가 가능합니다.

## 4. 최종 권고 (Final Recommendation)
현재의 t3.micro($9.64)에서 **t3.medium On-Demand($30.37)**로 먼저 전환하여 1주일간 부하를 테스트한 후, 시스템이 안정화되면 **1년 약정($17.75)**을 맺어 고정 지출을 최소화하는 방안을 추천드립니다.

---

> [!TIP]
> **Zero-Cost Tip**: AWS의 'Spot Instance'를 사용하면 최대 70% 이상 저렴할 수 있으나, 서버가 갑자기 중단될 리스크가 있어 추천 서비스용으로는 Reserved Instance가 가장 안전합니다.
