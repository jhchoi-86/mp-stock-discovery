## [v7.7.18] - 2026-04-05
### 🚩 상태: 로그인 502 에러 완전 박멸 및 레드팀 정밀 감사 완료
### 🛠 주요 변경 및 조치 사항
1. **502 에러 직접 원인 제거**:
    - `uuidv4` 선언 누락 및 `Queue`, `systemStatsService` 리콰이어 유실분을 모두 복구하여 프로세스 크래시를 차단했습니다.
2. **미들웨어 순서 교정 (로그인 및 CORS 해결)**:
    - CORS, BodyParser, CookieParser를 모든 라우터보다 앞에 배치하여 API 인증 및 데이터 파싱 무결성을 확보했습니다.
3. **KST 시간 및 데이터 중복 방지**:
    - Opening Range 필터의 KST 시간대 오차 수정 (Date Offset 적용).
    - CSV 임포트 시 동일 종목/타임프레임 중복 신호 누적 방지 로직 적용.
4. **텔레그램 알림 안정화**:
    - 야간 리포트 발송 시 4096자 초과 에러 방지를 위한 자동 분할 전송 시스템 도입.
5. **성능 최적화**:
    - `2M` 타임프레임 조회 기간 최적화 및 `emitProgress` 참조 오류 등 부차적 버그 총 12종 일괄 조치.

## [v7.7.4] - 2026-04-05
### 🚩 상태: 수동 동기화 점수 DB 저장 및 로직 표준화 완료
### 🛠 주요 변경 및 조치 사항
1. **점수 엔진 표준화 (`scoreEngine.cjs`)**:
    - 관리자 페이지 수동 동기화, 야간 자동 크론, 랜딩페이지 업데이트에서 제각각 사용되던 점수 계산 로직을 단일 모듈로 통합했습니다.
2. **수동 동기화 DB 저장 기능 추가**:
    - 이제 관리자 페이지에서 "동기화" 버튼을 클릭 시, 분석 직후 즉시 종목별 점수를 계산하여 `DailyStockSnapshot` DB 테이블에 기록합니다. (야간 크론과 동일한 수준의 데이터 영속성 확보)
3. **Sniper 정밀도 로직 반영**:
    - `scoreEngine`에 현재가와 2H 진입가 사이의 이격도를 평가하는 Sniper-Precision 로직이 포함되어 보다 정교한 필터링이 가능해졌습니다.

## [v7.7.2] - 2026-04-05
### 🚩 상태: 전 지점 가격 데이터 "원본 그대로" 완전 동기화 완료
### 🛠 주요 변경 및 조치 사항
1. **가격 보정 로직 전면 제거**: 
    - 텔레그램 메시지 및 랜딩페이지에서 상시 적용되던 "목표가 5% 상향 보정(Boost)" 로직을 삭제했습니다.
    - 이제 관리자 대시보드와 동일한 분석 시점의 원본 타겟 가격(`bb_upper`)이 노출됩니다.
2. **타임프레임 참조 통일**:
    - 텔레그램 리포트의 1차/2차 진입가 참조를 `1H/2H` 혼용에서 관리자 페이지와 동일한 **`2H` 고정(`result_2`, `result_3`)**으로 통일했습니다.
3. **용어 및 레이블 일치화**:
    - 랜딩페이지 UI의 레이블을 `(2H)`로 명시하여 분석 타임프레임의 투명성을 높였습니다.
